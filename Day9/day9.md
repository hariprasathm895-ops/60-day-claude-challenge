# Day 9 – NutriScope: Iterative App Building
## ABTalks 60-Day Claude AI Mastery Challenge

## What I Learned Today
Used Claude to build a full Nutrition Tracker app
in two iterations — MVP first, then Enhanced version.
This is called Iterative Prompting.

## Version 1 — MVP (Prompt 1)
- 20 foods database
- Basic food logging table
- Calorie + macro tracking
- Progress bars for nutrients
- Simple recommendations
- Dark theme UI with Chart.js

## Version 2 — Enhanced (Prompt 2)
- 60 foods database (3x more!)
- CSV upload feature
- 5-tab navigation (Dashboard, Log, Planner, Risk, Profile)
- 2-day meal planner
- Risk Analysis panel
- Radar chart + Doughnut chart
- Zinc and Magnesium tracking added
- Nutrition sources & educational disclaimer
- Health goal selector (lose/gain/maintain/muscle)

## Key Comparison
| Feature | MVP | Enhanced |
|---------|-----|----------|
| Foods | 20 | 60 |
| Charts | 1 | 3 |
| Nutrients tracked | 10 | 12 |
| Meal planner | No | Yes |
| Risk analysis | No | Yes |
| CSV upload | No | Yes |
| Navigation tabs | No | Yes |

## Key Learnings
1. Iterative prompting = build simple first, enhance next
2. Claude can build full SaaS-quality apps in one prompt
3. Version 2 is always better when built on Version 1
4. Single HTML file = no backend, zero cost deployment

## Key Takeaway
"Build fast. Enhance smart. Ship daily."
